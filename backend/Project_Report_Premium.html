1. Abstract

The Face Recognition Attendance System is a modern, automated solution designed to streamline the attendance-taking process in educational or corporate environments. By leveraging state-of-the-art computer vision and deep learning techniques, the system eliminates the inefficiencies of traditional manual attendance methods. Built with a robust FastAPI backend and a responsive Next.js frontend, the application provides real-time face detection, group recognition, and automated reporting.

2. Introduction

2.1 Problem Statement

Traditional methods of marking attendance, such as manual registers or card-based systems, are prone to human error, time-consuming, and susceptible to "buddy punching" or proxy attendance. In a classroom environment, significant productive time is wasted in calling out names, and physical sheets are hard to maintain over long periods.

2.2 Project Objective

Our primary goal was to develop a contactless, reliable, and efficient system that solves these issues. The core objectives included:

  • Biometric Accuracy: Automatically identify students using localized facial features.
  • Automation: Support bulk processing of student records with automated photo encoding via external URLs.
  • User Experience: Provide specialized dashboards for both Teachers (Administrators) and Students.
  • Privacy: Ensure security through role-based access control and branch-level isolation.

2.3 Scope

The scope of this project encompasses student registration, face encoding generation, live camera stream processing, automated attendance logging, and comprehensive analytics generation through specialized reporting modules.

3. Technology Stack

3.1 Backend (FastAPI)

The backend is built using FastAPI, a high-performance framework chosen for its asynchronous support and automatic JSON validation. SQLAlchemy handles ORM operations, allowing for robust interaction with the SQLite database.

3.2 Frontend (Next.js)

We used Next.js 14 for the frontend to ensure fast page loads and SEO optimization. Tailwind CSS was implemented for a modern, fluid user interface, while Lucide React provides the visual iconography.

3.3 Core AI/ML Libraries

The system's "brain" relies on face_recognition (built on dlib) which achieves near-human accuracy. OpenCV manages the webcam stream processing, and NumPy handles the multi-dimensional vector calculations.

4. System Architecture

6. Detailed Implementation

6.1 Face Encoding Process

During student registration or bulk import, the system executes a multi-step pipeline:

  • Downloads the source image from the provided link or upload.
  • Locates the primary face using HOG (Histogram of Oriented Gradients).
  • Extracts 68 facial landmarks to align the face.
  • Generates a unique 128-dimension vector (Biometric Signature).
  • Stores this vector in the database for future comparison.

7. Security and Privacy

We implemented a robust security model to protect student biometrics and attendance data:

  • JWT Protection: All system interactions require a valid JSON Web Token.
  • Branch Isolation: A CSE Teacher is mathematically restricted from viewing EE or CIVIL logs.
  • Signature Hash: We store the 128-dimensional vector, which is a mathematical representation. It is impossible to reconstruct the original image from this vector alone.
